# ブランチ戦略

〇〇PJでは**〇〇**を採用する。

〇〇PJでは以下の様にbranchを作成する。

- 本番（エンジニア以外動作確認）：main
- 動作確認（エンジニア、プロダクトオーナー）：dev
- 開発（アサインされたエンジニア）：feature/JIRA のチケット id_JIRA のチケット名
  - 例：feature/BLC-9\_フロント 〇〇画面の作成
